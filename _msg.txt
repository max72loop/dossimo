feat(cerfa): pre-remplissage de l'AH de l'oblige (Cas 1 - PDF a champs)

Donne le VRAI document : l'artisan televerse l'attestation sur l'honneur que
son oblige lui a remise ; si c'est un PDF a champs (AcroForm), Dossimo y reporte
les valeurs de la saisie unique et la rend pre-remplie, prete a verifier/signer.
Aucun calage par oblige : le mapping des champs est fait par le LLM a partir du
nom technique + du libelle (tooltip /TU) de chaque champ.

Garde-fou central : on n'ecrit jamais de valeur inventee. Le prompt interdit
toute correspondance incertaine, on ne conserve que les mappings >= 0.7 vers un
champ reellement present, et les champs propres a l'oblige (n° de dossier, etc.)
restent vides. Traitement en memoire : le PDF source n'est pas conserve.

- acroform-inspect.ts : enumere les champs (nom, type, tooltip, options).
- oblige-fill.ts : canonicalFacts (faits du dossier), mapAhFields (mapping LLM),
  fillUploadedAh (remplissage par type + aplatissement, non bloquant).
- oblige-actions.ts : action auth-scopee remplirAhOblige (validation PDF/taille,
  detection AcroForm, renvoie le PDF pre-rempli en base64 + recap).
- composant client ah-oblige-fill : upload -> telechargement + champs restants.
- UI dossier : bloc propose sous le formulaire officiel (cote CEE).

Valide en banc d'essai : 17/19 champs remplis sur une AH synthetique, les 2
champs non connaissables laisses vides, case a cocher geree, PDF aplati OK.
